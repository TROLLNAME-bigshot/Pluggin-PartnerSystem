# Отчет по Контрольной точке №2: Ядро и Инфраструктура

## 1. Текущий статус разработки ядра

На текущем этапе в проекте **PromoDawn** реализовано основное ядро партнерской системы для Minecraft-сервера.

### Готовые ключевые функции бизнес-логики

- Реализована система **активации партнерских промокодов** игроками на сервере.
- Реализовано **сохранение связи игрока с конкретным партнером** после активации промокода.
- Реализовано **хранение промокодов, владельцев, процентов вознаграждения и статистики использования**.
- Реализована обработка **донат-транзакций игроков**, связанных с партнерскими промокодами.
- Реализован **автоматический расчет заработка партнера** на основе процента от суммы доната.
- Реализовано **сохранение транзакций, выплат и логов операций** в MySQL.
- Реализована возможность **отметки выплат партнерам** и подсчета общей выплаченной и невыплаченной суммы.
- Реализован **игровой интерфейс просмотра статистики** и дохода партнера прямо в Minecraft.
- Реализованы команды администратора для **добавления транзакций, проверки данных и управления партнерской системой**.
- Реализована архитектурная база для интеграции с внешним сайтом через передачу события покупки в игровой сервер.

Таким образом, базовая бизнес-логика продукта уже написана и работает: от активации промокода игроком до фиксации доната и отображения партнеру его заработка в игре.

## 2. Интеграция стандартов обслуживания (из 1-й части практики)

### Изоляция секретов

Для безопасной публикации репозитория конфигурация `config.yml` подготовлена к отправке без реальных секретов: вместо настоящего пароля базы данных используется шаблонное значение `CHANGE_ME`.

Ниже приведен пример безопасного файла `.env.example`, который рекомендуется использовать в дальнейшем для локальной и серверной настройки:

```env
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_DATABASE=minecraft_plugins
MYSQL_USER=your_mysql_user
MYSQL_PASSWORD=your_mysql_password
```

В публичный репозиторий отправляется только безопасная версия конфигурации без действующих учетных данных.

### Логирование

В проекте реализовано логирование основных операций партнерской системы.

Используемые подходы:
- логирование событий активации промокодов;
- логирование транзакций донатов;
- логирование начислений партнерам;
- логирование выплат и административных действий;
- сохранение журналов операций в базе данных через таблицу `partner_logs`.

На уровне архитектуры логирование инициализируется через стандартные механизмы Java/Spigot-плагина и дополняется журналированием бизнес-событий в базе данных.

## 4. Состав реализованного ядра по модулям

### Основные модули проекта

- `PromoDawn.java` — точка входа плагина и регистрация сервисов/команд.
- `PromoService.java` — логика промокодов, привязка игрока к партнеру, статистика использования.
- `PartnerFinanceService.java` — учет донатов, начисление дохода партнерам, выплаты, финансовая статистика.
- `DatabaseManager.java` — подключение к MySQL и инициализация таблиц.
- `PartnerCommand.java` — административная обработка транзакций и выплат.
- `MyPromoCommand.java`, `PromosCommand.java`, `GuiBuilder.java` — отображение статистики и доходов внутри игры.

### Таблицы данных

В проекте предусмотрены таблицы для хранения:
- использований промокодов;
- транзакций партнерской системы;
- выплат партнерам;
- общей выплаченной суммы;
- журналов действий и операций.
